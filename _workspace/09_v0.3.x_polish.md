## C1. ShippingAddressForm 사용법

```tsx
import { ShippingAddressForm } from '@/components/commerce/ShippingAddressForm';

// 제출 버튼 내장 버전 (마이페이지 배송지 관리 등)
<ShippingAddressForm
  initialValue={savedAddress}
  onSubmit={(addr) => saveAddress(addr)}
/>

// 상위에서 유효성만 관찰 (체크아웃에서 "결제" 버튼이 제출을 대신함)
<ShippingAddressForm
  initialValue={draftAddress}
  showSubmit={false}
  onValidChange={setDraftAddress} // ShippingAddress | null
/>
```

### 필드
| # | 필드 | 타입 | 필수 | zod 규칙 |
|---|------|------|------|----------|
| 1 | `recipientName` | string | ✅ | `z.string().min(1)` |
| 2 | `phone` | string | ✅ | `z.string().min(1)` |
| 3 | `zipCode` | string | ✅ | `z.string().min(1)` |
| 4 | `address1` | string | ✅ | `z.string().min(1)` (도로명) |
| 5 | `address2` | string | ❌ | `z.string().default('')` (상세) |
| 6 | `memo` | string | ❌ | `z.string().optional()` |

총 6개 필드 · 4개 필수 · 2개 선택. `shared-types` `ShippingAddressSchema` 와 1:1.

### 상호작용
- 모바일 1열 / `md:` 이상 2열 grid (우편번호·도로명·상세·메모는 full-width span)
- `onValidChange` 는 zod parse 성공 시 값, 실패 시 `null` 전달 → 상위 버튼 disabled 근거
- `handleSubmit` 시 `ShippingAddressSchema.safeParse` → 실패 시 필드별 에러 한국어 메시지 표시 (`friendly()`)
- 우편번호 "주소 검색" 버튼: 아직 `setShowSearchNotice(true)` 로 "TODO v0.4" 안내만 노출. 수동 입력 유지.

### 저장
- `stores/shipping.ts` (`useShippingStore`) — localStorage `nuxia2-shipping` 로 `savedAddress` persist.
- 체크아웃 진입 시 저장된 주소 있으면 `AddressSummary` 요약 카드 + "수정" 버튼. 편집 모드에서 결제 누르면 자동 저장.

## C2. ProductFilterBar 사용법

```tsx
<ProductFilterBar
  keyword={keyword}
  categoryName={categoryName}     // string | null (null = 전체)
  sort={sort}                     // 'newest' | 'popular' | 'priceAsc' | 'priceDesc'
  onKeywordChange={(v) => updateQuery({ q: v })}
  onCategoryChange={(v) => updateQuery({ categoryName: v })}
  onSortChange={(v) => updateQuery({ sortBy: v })}
/>
```

### 카테고리 칩 (6개)
전체 · 의류 · 가방 · 신발 · 액세서리 · 라이프

### 정렬 옵션 (4개)
| value | 라벨 | 동작 |
|-------|------|------|
| `newest` | 최신순 | `createdAt desc` |
| `popular` | 인기순 | `popularity desc` (기본) |
| `priceAsc` | 낮은가격순 | `salePriceKrw asc` |
| `priceDesc` | 높은가격순 | `salePriceKrw desc` |

### URL 파라미터
- `?q=` (검색어), `?categoryName=` (카테고리), `?sortBy=` (정렬)
- `router.replace(..., { scroll: false })` 로 스크롤 위치 유지
- 검색은 로컬 상태에 입력 즉시 반영, 300ms debounce 후 URL 갱신
- 뒤로가기 시 URL → 상태 재hydrate (`useEffect`)

### Mock → API 전환 시점 (v0.4)
```ts
// 현재
const products = filterMockProducts({ categoryName, keyword, sort });

// 목표
const { data } = useQuery(['products', ...],
  () => api.get('/products', ProductListSchema, {
    params: { categoryName, keyword, sort, cursor },
  })
);
```

## C3. PaymentMethodPicker 사용법

```tsx
const [method, setMethod] = React.useState<PaymentMethod>('card');

<PaymentMethodPicker value={method} onChange={setMethod} />

// 결제 호출 시
await requestPayment({
  ...,
  channelKey: CHANNEL_KEY_MAP[method],
});
```

### 결제 방법 (3개)
| key | 탭 라벨 | 설명 |
|-----|--------|------|
| `card` | 카드 | 신용/체크카드 결제 (할부 지원) |
| `transfer` | 계좌이체 | 실시간 계좌이체 / 가상계좌 발급 |
| `easypay` | 간편결제 | 토스페이 · 네이버페이 · 카카오페이 |

### channelKey 매핑 (체크아웃 페이지 내)
```ts
const CHANNEL_KEY_MAP: Record<PaymentMethod, string> = {
  card:     process.env.NEXT_PUBLIC_PORTONE_CARD_CHANNEL_KEY     ?? 'channel-key-card',
  transfer: process.env.NEXT_PUBLIC_PORTONE_TRANSFER_CHANNEL_KEY ?? 'channel-key-transfer',
  easypay:  process.env.NEXT_PUBLIC_PORTONE_EASYPAY_CHANNEL_KEY  ?? 'channel-key-easypay',
};
```

- 포트원 콘솔에서 채널 3개 (card / transfer / easypay)를 별도 등록 후 해당 키를 `.env.local` 에 주입.
- 미등록 시 fallback 문자열 (`channel-key-*`)은 데모 전용이며 실제 결제 시 포트원 SDK 에서 실패.
- `apps/web/lib/portone.ts` 의 `requestPayment` 는 이미 `params.channelKey ?? NEXT_PUBLIC_PORTONE_CHANNEL_KEY` 구조 → 수정 불필요.
