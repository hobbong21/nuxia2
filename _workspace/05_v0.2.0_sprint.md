v0.1.0의 **"구조적 기반"**에서 v0.2.0은 **"로컬 실행 가능 + 자동화 검증"**으로 이동한다.

## 범위 (Must)

### M1. 실제 Prisma 마이그레이션
- `apps/api/prisma/migrations/0_init/migration.sql` 생성
- Prisma CLI로 생성된 DDL 스냅샷 (스키마 16 모델 + 9 enum 전부)
- `apps/api/prisma/seed.ts` — 4대 체인 + 상품 10종 + 주문 시뮬레이션 3종
- `package.json prisma.seed` 필드 연결

### M2. Docker Compose 로컬 환경
- `docker-compose.yml` 루트에 생성
  - `postgres` 서비스 (포트 5432, 볼륨 마운트, healthcheck)
  - `redis` 서비스 (포트 6379, healthcheck)
  - `.env.docker` 샘플
- 선택적으로 `api`, `web` 서비스 (dev profile)
- `docs/QUICKSTART.md` 업데이트: `docker compose up -d` 한 줄로 시작

### M3. E2E 테스트 스위트 (실제 실행 가능)
- `scripts/qa/` 하위에 vitest 기반 테스트 파일
  - `setup-fixtures.ts` — 4대 체인 A→B→C→D 시드
  - `referral-1m-test.ts` — 1,000,000원 기준 시나리오 (C=30k/B=50k/A=170k/합 250k)
  - `refund-revert-test.ts` — 전체/부분 환불 역정산
  - `abuse-self-referral.ts` — A1-direct + A1-ancestor
  - `abuse-circular.ts` — A2 순환참조 차단
  - `abuse-multi-account.ts` — A3 + T5 탈퇴 쿨다운 + T6 STAFF
  - `payment-mismatch.ts` — 포트원 금액 불일치 거절
  - `boundary-shape-check.ts` — API 응답 × shared-types zod 매칭
  - `run-all.ts` — 전체 시나리오 오케스트레이션
- 루트 `vitest.config.ts` + root `scripts/qa/package.json` 또는 apps/api의 test 스크립트로 통합

### M4. 잔여 non-blocker 3건 해소
- **N1**: `apps/api`에 `@nuxia2/shared-types` workspace dep 추가 + import 경로 정리. 로컬 복제된 `ShippingAddressSchema` 제거.
- **N2**: `ProductListQuerySchema.categoryId` → `categoryName` 리네임. Prisma/서비스/컨트롤러 전체 반영.
- **N3**: checkout/success 재시도 버튼을 위한 `POST /payments/orders/:orderId/confirm` idempotent 가드 강화 — 이미 `PAID`면 `{ok:true, alreadyPaid:true}` 반환, 레퍼럴 재분배 생략.
