## 2FA 흐름

1. 사용자 로그인 (TOTP 미활성) → 기존과 동일 `{ accessToken, refreshToken, user, role }`
2. `POST /auth/2fa/setup` (Bearer): `{ otpauthUrl, qrDataUri, secret }`. DB 에 암호화 저장, `totpEnabled=false`.
3. 사용자가 Authenticator 에 등록 후 6자리 코드 입력 → `POST /auth/2fa/verify { code }`. 성공 시 `totpEnabled=true`.
4. 다음 로그인: `POST /auth/login { email, password }` → `{ needsTotp: true, userId }` 응답.
5. 2단계: `POST /auth/2fa/login { userId, code }` → `{ accessToken, refreshToken, user, role }`.
6. `POST /auth/2fa/disable { code }` → 비활성 + secret null.
