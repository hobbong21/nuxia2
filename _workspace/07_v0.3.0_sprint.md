## 범위 (Must)

### M1. 관리자 화면 skeleton (frontend-engineer)

v0.2의 Admin API를 UI로 소비하는 최소 기능:
- **라우트 그룹**: `apps/web/app/(admin)/` 
  - `/admin/abuse-logs` — 테이블 + kind 필터 + 무한 스크롤(cursor)
  - `/admin/users` — 검색 + 목록
  - `/admin/users/[id]` — 사용자 상세 + 3세대 트리 시각화 + 플래그/미성년 해제 버튼
  - `/admin/payouts` — 정산 내역 (기존 API 재활용)
- **가드**: `role === 'ADMIN'`만 접근. 비관리자는 `/` 리다이렉트
- **디자인**: `_workspace/02_designer_spec.md` 토큰 재사용, 관리자 톤 (밀도 높은 테이블, 필터 칩)
- **상태 UI**: 어뷰징 로그에 `kind` 색상 코딩 (`SELF_REFERRAL` 빨강, `CIRCULAR` 주황, `DUPLICATE_CI` 노랑, `STAFF_REFERRAL_FORBIDDEN` 회색, `WITHDRAW_REJOIN_COOLDOWN` 보라)
- **트리 시각화**: 기존 `ReferralTreeNode` 재사용 + 관리자 전용 "플래그" 오버레이

### M2. Dockerfile + compose full profile (infra-engineer)

- `apps/api/Dockerfile` — multi-stage (base → build → runtime), `node:20-alpine`
  - build stage: `pnpm install --frozen-lockfile` + `pnpm --filter @nuxia2/api build`
  - runtime: dist 복사 + `pnpm exec prisma generate` + `CMD ["node", "dist/main.js"]`
  - non-root user
- `apps/web/Dockerfile` — Next.js standalone 출력 기반 multi-stage
  - `next.config.mjs`에 `output: 'standalone'` 추가 필요
  - runtime은 standalone + static + public 복사
- `.dockerignore` 각 앱에 추가
- `docker-compose.yml`의 `full` profile 활성화 확인 (의존성 + env 경로)

### M3. GitHub Actions CI (infra-engineer)

- `.github/workflows/ci.yml`:
  - trigger: `push: [main]`, `pull_request: [main]`
  - matrix: Node 20
  - steps:
    1. Checkout
    2. Setup pnpm 9
    3. `pnpm install --frozen-lockfile`
    4. `pnpm -r run typecheck` (shared-types + web + api)
    5. `pnpm -r run lint` (api, web)
    6. Services: `postgres:16-alpine`, `redis:7-alpine` (services 블록)
    7. `pnpm --filter @nuxia2/api exec prisma migrate deploy`
    8. `pnpm --filter @nuxia2/api test:e2e`
- `.github/workflows/docker-build.yml` (optional):
  - Push to main → buildx → multi-arch amd64+arm64 → GHCR push

### M4. 마지막 non-blocker 해소 (backend-engineer)

- **Webhook E2E 테스트** — `apps/api/test/webhook-portone.test.ts` 추가
  - HMAC 서명 생성 → POST `/webhooks/portone` → 200 OK + WebhookEvent 생성
  - 잘못된 서명 → 401
  - 동일 event 재전송 → idempotent (두 번째 POST는 no-op, WebhookEvent 중복 없음)
- **categoryId legacy 제거** — `product.controller.ts` 에서 legacy 분기 제거, shared-types와 백엔드 완전 일치
