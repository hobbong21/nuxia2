# 11c. v0.4 Infra 변경 요약

> 담당: infra-engineer
> 기반: `_workspace/10_v0.4.0_sprint.md`

## 변경 요약

v0.4에서 backend가 도입할 3가지 신규 인프라 요소(admin API key / metrics internal secret / 2FA TOTP)와 frontend의 mock gate 전환을 지원하는 환경변수·CI·docker-compose·문서 업데이트.

## 수정 파일 목록

| 파일 | 변경 내용 |
|------|----------|
| `apps/api/.env.example` | v0.4 섹션 append (`ADMIN_API_KEY`, `METRICS_INTERNAL_SECRET`, `TOTP_ISSUER` + FE mock 주석) |
| `.github/workflows/ci.yml` | e2e job env에 3개 테스트용 변수 추가 |
| `docker-compose.yml` | api 서비스에 3개 env, web 서비스에 `NEXT_PUBLIC_USE_MOCK` 추가 |
| `.env.docker.example` | docker compose용 4개 변수 기본값 추가 |
| `docs/QUICKSTART.md` | "v0.4 새 환경 변수" 섹션 신규 (BE/FE/2FA/metrics 가이드) |
| `README.md` | 핵심 기능에 2FA/metrics/audit log 3개 불릿 추가 |

## 실행 가이드

### 1. 로컬 개발 (도커 미사용)
```bash
cp apps/api/.env.example apps/api/.env.local
# 필요 시 편집:
#   ADMIN_API_KEY=my_dev_admin_key          (설정하면 /admin/* 가드 강화)
#   METRICS_INTERNAL_SECRET=                (로컬은 비워도 /metrics 공개)
#   TOTP_ISSUER=Nuxia2-Local
```

### 2. Docker Compose 풀 스택
```bash
cp .env.docker.example .env
# 필요 시 .env 에서 ADMIN_API_KEY, METRICS_INTERNAL_SECRET, TOTP_ISSUER 커스텀
docker compose --profile full up -d
```
- api 컨테이너는 `ADMIN_API_KEY=dev_admin_api_key`, `METRICS_INTERNAL_SECRET=dev_metrics_secret`, `TOTP_ISSUER=Nuxia2-Dev` 기본값 사용
- web 컨테이너는 `NEXT_PUBLIC_USE_MOCK=0` 기본 (실 API 호출)

### 3. CI
- `.github/workflows/ci.yml` e2e job에 `ADMIN_API_KEY=test_admin_api_key_v4`, `METRICS_INTERNAL_SECRET=test_metrics_internal_secret`, `TOTP_ISSUER=Nuxia2-Test` 주입
- 기존 `prisma migrate deploy`가 `migrations/*` 전체를 배포하므로, backend가 추가할 `20260421000001_add_totp` (또는 유사 타임스탬프) 마이그레이션도 자동 포함 — 워크플로우 수정 불필요.

## package.json 확인 (쓰기 금지)
