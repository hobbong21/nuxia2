## M4-1: Webhook 테스트 설계 근거

`PortoneWebhookController` 를 직접 `new` 해서 호출한다. Nest HTTP 서버 부팅/supertest 도입은 다음 이유로 유보:

1. 기존 테스트 스위트가 모두 서비스 직접 호출 패턴이며, `INestApplication` 부팅은 PaymentModule → ReferralModule → BullMQ(ioredis) 까지 의존성 그래프를 끌어올려 DB 외 외부 의존(redis)이 필요해진다.
2. 컨트롤러의 HMAC 검증 / DB insert / idempotency 분기는 전부 동기 메서드 호출 범위에 있어 HTTP 계층 없이도 100% 커버 가능.
3. `PaymentService` 는 dispatch 경로에서 호출되지 않으므로(현재 구현은 `this.logger.log` 만 수행) `NoopPaymentService` 스텁으로 주입.

### HMAC 규격

컨트롤러 `verifyHmac()` 와 완전 일치:

```
signature = hex(hmac-sha256(PORTONE_WEBHOOK_SECRET, rawBody))
```

- `rawBody = JSON.stringify(body)` (컨트롤러가 body 가 string 이 아니면 stringify)
- `sha256=` prefix 허용 (테스트는 prefix 없이 hex 만 전송)
- `Buffer.from(sig, 'hex')` 길이 비교 후 `timingSafeEqual`

### 검증 항목 (it 블록 6개, 목표 5+ 충족)

| # | 시나리오 | 기대 |
|---|---------|------|
| 1 | 유효 서명 | `{ ok: true }`, WebhookEvent 1건, `signatureOk=true`, `processedAt !== null` |
| 2 | 잘못된 서명 | `BadRequestException`, WebhookEvent 0건 |
| 3 | 동일 (source, externalId, eventType, eventTimestamp) 재전송 | 2회차 `{ ok: true, duplicate: true }`, DB 행은 1건 (P2002 흡수) |
| 4 | 동일 paymentId + 다른 eventType (Paid / Cancelled) | 각각 별개 행 2건 |
| 5 | `ALLOW_UNSIGNED_WEBHOOK=1` + `NODE_ENV=development` + 서명 누락 | 200 OK, WebhookEvent 1건 |
| 6 | `ALLOW_UNSIGNED_WEBHOOK=1` + `NODE_ENV=production` + 서명 누락 | 여전히 거부 (토글이 프로덕션에서는 무시됨) |

각 테스트는 `beforeEach` 에서 `prisma.webhookEvent.deleteMany({})` 로 테이블을 비우고 env 를 기본값으로 리셋한다. `afterAll` 에서는 원본 env 를 복원.

## S1: 로그 관찰성

### logger.config.ts
- `NODE_ENV==='production'` → plain JSON (pino 기본)
- 그 외 → `pino-pretty` transport (싱글라인, 컬러, `SYS:HH:MM:ss.l`)
- `pino-pretty` 미설치 시 자동 JSON fallback (`require.resolve` try/catch)
- `genReqId`: ALS store 값 우선 → 헤더 → `crypto.randomUUID()`
- `customProps`: 모든 로그 라인에 `correlationId` 필드 주입
- `customLogLevel`: 4xx→warn, 5xx/error→error, 나머지 info

### correlation-id.interceptor.ts
- 글로벌 인터셉터로 주입 (`main.ts`). `LoggingInterceptor` 보다 먼저 실행돼 로그에 id 가 포함되게 한다.
- `x-correlation-id` 헤더 (있으면 재사용, 없으면 `randomUUID()`)
- `correlationIdStore.run({ correlationId }, ...)` 로 AsyncLocalStorage 스코프 열기
- 응답 헤더에도 동일 id echo

### 미도입 (의도적)
- `nanoid`: Node 18+ 내장 `crypto.randomUUID()` 로 대체 → 신규 의존성 0.
- `AuditLog` vs `AbuseLog` 강화: 스키마는 v0.2 에서 이미 분리되어 있음 (`model AuditLog` / `model AbuseLog`). 추가 래퍼는 v0.4 로 유보.
