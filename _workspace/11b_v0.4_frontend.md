## 2FA 흐름 단계

### Setup (마이페이지 → 2FA 설정)

1. `/mypage/security` 진입 → `GET /auth/2fa/status`
2. "2단계 인증 설정" 클릭 → 모달 open (step=intro)
3. "시작하기" → `POST /auth/2fa/setup` → QR 데이터 URI 표시 (step=qr)
4. "다음" → 6자리 입력 필드 (step=verify)
5. 6자리 완성 → `POST /auth/2fa/verify` (자동 제출)
6. 성공 → 토스트 + 모달 close + 상태 refresh

### Disable

1. `/mypage/security` → "비활성화" 클릭 → 모달 open
2. 현재 TOTP 6자리 입력 → 자동 제출 → `POST /auth/2fa/disable`
3. 성공 → 토스트 + 모달 close + 상태 refresh

### Login 2단계

1. 이메일/비밀번호 submit → `POST /auth/login`
2. 응답 분기:
   - `AuthResponse` → 홈 이동
   - `{ needsTotp: true, userId }` → stage=totp
3. 6자리 입력 → 자동 제출 → `POST /auth/2fa/login { userId, code }`
4. 성공 → 토스트 + 홈 이동 (`router.refresh()` 로 서버 컴포넌트 재페치)

총 **4 흐름**: setup / verify / disable / login-2nd-step

**총 safeParse 호출: 5** (getUserTree 는 `TreeNodeSchema` import 대기 중 — TODO 주석).

실패 시 동작:
- 스키마 mismatch → 콘솔 에러 + 빈 목록/기본값 반환
- HTTP 4xx/5xx → 예외 → 호출부 catch → 빈 목록 + 콘솔 에러
- 호출부(각 admin page) 는 zero-state UI 자연 렌더링

- **반응형**: `max-w-md mx-auto` (360px~) + `px-base` 기반, 모바일 퍼스트
