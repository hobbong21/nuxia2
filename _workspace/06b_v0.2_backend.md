## 변경 요약

### M1. Prisma 마이그레이션 + 시드
- `apps/api/prisma/migrations/20260421000000_init/migration.sql` (약 360 라인)
  - enum 9종 `CREATE TYPE`
  - 테이블 16개 `CREATE TABLE`
  - UNIQUE 인덱스 10건 (`User.email`, `User.ciHash`, `User.referralCode`, `BankAccount.userId`, `Product.slug`, `Cart.userId`, `CartItem(cartId,productId)`, `Order.paymentId`, `ReferralLedger 4-tuple`, `Payout(userId,periodStart,periodEnd)`, `WebhookEvent 4-tuple`)
  - 일반 인덱스 17건
  - 외래키 15건 (User self-ref, Session, BankAccount, CartItem x2, Order, OrderItem x2, Refund, ReferralLedger x2, Payout x2, AbuseLog, AuditLog)
- `apps/api/prisma/migrations/migration_lock.toml` — `provider = "postgresql"`
- `apps/api/prisma/seed.ts` — 시드 스크립트
  - 4대 체인 A(root) → B(gen1) → C(gen2) → D(gen3), 모두 ACTIVE + ciHash 유니크
  - STAFF 계정 1명 (T6 검증용, referralCode=STAFF001)
  - 상품 10종 (5,000~500,000원, placehold.co 이미지)
  - 주문 3건 (order1=D PAID 100k, order2=C PENDING 49k, order3=B PAID 200k×2 items)
  - `PayoutTaxConfig` 1건 (withholdingBps=330, kind=BUSINESS_INCOME, effectiveFrom=2026-01-01)
  - `process.env.NODE_ENV === 'production'` 시 즉시 throw (실수 방지)
  - `APP_ENCRYPTION_KEY` / `APP_ENCRYPTION_SALT` 미설정 시 개발용 고정 값 주입
- `apps/api/package.json`
  - `scripts.prisma:seed` 추가
  - 루트 `"prisma": { "seed": "tsx prisma/seed.ts" }` 필드 추가
  - `dependencies` 에 `"@nuxia2/shared-types": "workspace:*"` 추가

### M4-N2. category 리네임 (categoryId → categoryName 일관성)
- `packages/shared-types/src/product.ts`
  - `ProductListQuerySchema.categoryId: IdSchema.optional()` → `categoryName: z.string().optional()`
- `apps/api/src/modules/product/product.service.ts`
  - `ListProductQuery.category` → `categoryName`
  - Prisma `where: { category: { equals: query.categoryName } }` 로 매핑
- `apps/api/src/modules/product/product.controller.ts`
  - `@Query('categoryName')` 신규 + 하위호환 `@Query('category')` fallback

### M4-N3. Payment confirm idempotent 가드 강화
- `packages/shared-types/src/payment.ts`
  - `PaymentConfirmResponseSchema` 에 `alreadyPaid: z.boolean().optional()` 추가
- `apps/api/src/modules/payment/payment.service.ts`
  - `confirm()` 진입부:
    - `order.status===PAID && order.paymentId===paymentId` → `alreadyPaid:true` 응답 (레퍼럴 재분배 생략)
    - `order.status===PAID && order.paymentId!==paymentId` → 409 `PAYMENT_ID_MISMATCH`
  - `buildConfirmResponse(order, message, alreadyPaid=false)` — 옵셔널 필드 조건부 포함

## 실행 절차

```bash
# 1. 의존성 설치 (workspace 링크)
pnpm install

# 2. PG/Redis 준비 (infra-engineer 의 docker-compose 로 기동)
docker compose up -d postgres redis

# 3. 마이그레이션 적용
cd apps/api
pnpm prisma migrate deploy    # prod-like. 개발 중에는 `prisma migrate dev` 로 자동 생성 가능

# 4. Prisma client 재생성
pnpm prisma:generate

# 5. 시드
pnpm prisma:seed
#  또는
pnpm exec prisma db seed

# 6. 앱 기동
pnpm dev
```

### 환경변수 요구
- `DATABASE_URL` — Postgres 접속 (docker-compose 기본값 호환)
- `JWT_SECRET` — ≥32 chars
- `APP_ENCRYPTION_KEY` — 64 hex chars (32 bytes)
- `APP_ENCRYPTION_SALT` — 임의 문자열
- 선택: `JWT_ACCESS_TTL=2h`, `JWT_REFRESH_TTL=14d`, `HOLD_DAYS=7`, `WITHDRAW_COOLDOWN_DAYS=30`
