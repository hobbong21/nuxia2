## 라우트 (5개)

| Path                        | 구성                                        |
|-----------------------------|---------------------------------------------|
| `/admin`                    | KPI 4장 (차단수/미지급/미성년/활성)         |
| `/admin/abuse-logs`         | DataTable + kind 5종 필터 칩 + cursor       |
| `/admin/users`              | 검색 폼 + DataTable + cursor                |
| `/admin/users/[id]`         | 프로필 KV 4장 + UserTreePanel + 플래그/해제 |
| `/admin/payouts`            | DataTable (gross/tax/net/상태) + cursor     |

## 컴포넌트 (7개)

1. **Sidebar** — 4항목 내비게이션, active 상태, 다크 토큰
2. **DataTable<T>** — `columns[]`, 홀짝행, 숫자 우측정렬, 빈상태, toolbar, 커서 nextHref
3. **AbuseKindBadge** — kind 5종에 색+아이콘+라벨 3중 인코딩 (designer_spec §8-2)
4. **AdminKpiCard** — earnings-xl 숫자 + 서브 라벨 + tone (default/warning/danger/success)
5. **UserTreePanel** — 기존 `ReferralTreeNode` 를 래핑. 플래그된 노드 요약 + 빨간 점 범례
6. **FlagUserButton** — native `<dialog>` 모달, 사유 입력(필수), `adminApi.flagUser` 호출
7. **ReleaseMinorButton** — MINOR_HOLD 일 때만 활성, `adminApi.releaseMinor` 후 reload

## Role guard 전략

`apps/web/middleware.ts` (신규):

- matcher: `/admin/:path*`
- 쿠키 `nx_role === 'ADMIN'` 이면 통과, 아니면 `/?reason=admin_required` 로 redirect
- `NEXT_PUBLIC_ADMIN_BYPASS=1` 이면서 `NODE_ENV !== 'production'` 일 때만 우회
  → 개발 편의용. 프로덕션 빌드는 항상 guard 적용
- **TODO(auth):** v0.4 에서 실 JWT 세션 통합 (현재는 쿠키 기반 mock; auth store 미도입)

## API Client 확장

`apps/web/lib/admin-client.ts` 에 7개 메서드 추가:

| 메서드                      | 경로 계약 (03b)                        | 목 동작                           |
|-----------------------------|---------------------------------------|-----------------------------------|
| `getKpi()`                  | (신규, BE 제안 필요)                  | 고정 KPI                          |
| `getAbuseLogs({kind,cursor,limit})` | `GET /admin/abuse-logs`       | 20건 인메모리 필터                |
| `getUsers({query,cursor,limit})`    | `GET /admin/users` (신규)     | 10건 인메모리 검색                |
| `getUser(id)`                       | `GET /admin/users/:id`(신규)  | 단건 조회                         |
| `getUserTree(id)`                   | `GET /admin/users/:id/tree`(신규) | `MOCK_DASHBOARD.tree` 반환    |
| `getPayouts({cursor,limit})`        | `GET /admin/payouts` (신규)   | 5건                               |
| `flagUser(id, {flagged,reason})`    | `POST /admin/users/:id/flag`(신규) | in-memory mutation           |
| `releaseMinor(id)`                  | `POST /admin/users/:id/release-minor` | MINOR_HOLD→ACTIVE 전이   |

**USE_MOCK 플래그** (`true` 고정): 실 BE 연동 시 `api.get/post` 호출로 스왑 후 해제.

## Mock 데이터 규모

| 종류         | 레코드 수 | 비고 |
|--------------|-----------|------|
| AbuseLog     | **20건**  | kind 5종 × 4건 균등 분포 |
| User         | **10건**  | STAFF 1, MINOR_HOLD 1, ACTIVE 8 (flagged 2) |
| Payout       | **5건**   | PAID 2 / PENDING 1 / WITHHELD 1 / CLAWBACK 1 |
| Tree         | 1개 (3뎁스) | `MOCK_DASHBOARD.tree` 재사용 (셀프레퍼럴 차단 노드 포함) |
| KPI          | 고정      | blockedThisMonth=20 · pendingPayouts=488,005원 · minorHold=1 · active=8 |

## 디자인 규약 적용

- `(admin)/layout.tsx` 루트에 `className="dark"` 부여 → 기본 다크 테마
- DataTable: `h-10` 행, 홀짝 `bg-zinc-950/60`, 숫자 `tabular-nums` + 우측 정렬
- 금액은 전부 기존 `formatKrw` (BigIntString 안전)
- AbuseKindBadge: 색(배경/테두리) + 아이콘 + 라벨 3중 인코딩
- 터치 타겟: 사이드바 링크 `h-10`, 칩 `h-7`, 버튼은 shadcn `size` 토큰 그대로
